## Lec7 Value-based RL 即 Q-learning
1. 我们直接把 actor-critic 算法的 a 设定为选择 Q(s, a) 最大的 a(概率为 1，其他概率为 0). 这样就不用训练 actor 了，直接去除了 Policy-Gradient，因此可以使用离线 replay-buffer，只要 (s, a) 非常丰富并且 Q 拟合得够好就能 work.
2. 可能会遇到贪心永远走老路的问题，因此可使用 epsilon-greedy（1 - epsilon 概率随机动作）或者 boltzmann 探索（按照 Q 大小选择动作）
3. 这种 Q-learning 仅支持离线，因为你只有 Q(s, a)，离散动作下枚举即可获得最优动作，连续动作下你会无法求出哪个 a 是最优的，连动作都没法选.
4. 这种写法不 work（根据目标函数更新完 Q 参数也相当于改变了目标函数，导致在目标附近来回震荡），我们来看下一章如何优化.

## Lec8 Q-learning in practice
1. DQN's target-network 方法: 复制一份 Q2 目标网络不更新参数只算 y = r(s, a) + gamma * Q2(s', a')，每隔 10000 步同步一次（或者  polyak 方法：每次按 0.001 缓慢更新权重）. 原来的网络 Q1 则与环境互动并存数据.
    - 通过离线 replay buffer 采样更好，因为在线交互的 (s, a) 分布与 Q 相关性太强，容易导致训练震荡.
2. n-step 技巧：连续计算后面 N 步奖励衰减求和。
3. 逐渐降 epsilon 探索率.
4. huber 损失：误差小时依然用 mse，误差大时改用绝对值.
5. 更强的方法：Double DQN. 用的仍然是原来两套网络.
   ```python
   # DQN
   # 从 replay buffer 中采样多个 (s, a, r, s_next, done)
   a_next = argmax_a Q_target(s_next, all a)
   y = r + gamma * Q_target(s_next, a_next) # Q_target 方差会导致 y 期望增高，Q_online 对该动作的期望也偏高，后续还更可能选它
   loss = mse(Q_online(s, a), detach(y))

   # Double DQN
   a_next = argmax_a Q_online(s_next, all a)      # 训练网络选动作
   y = r + gamma * Q_target(s_next, a_next)   # 目标网络评估动作. 此时 Q_online 的方差不会引起 y 期望偏高
   loss = mse(Q_online(s, a), detach(y))
   ```
> 在传统的 DQN 算法中，本来就存在两套函数的神经网络——目标网络和训练网络（参见 7.3.2 节），只不过的计算只用到了其中的目标网络，那么我们恰好可以直接将训练网络作为 Double DQN 算法中的第一套神经网络来选取动作. (- 动手学强化学习)

> 即使 Q_online 因正误差选中了某个动作，Q_target 对同一个动作的估计不一定也正好虚高，所以 target 不会系统性地总取到最大正误差。(- GPT5.5)

6. DDPG: 结合 Soft Actor-Critic 和 DQN（分离评分网络和被更新网络），一共有四个网络. 但由于 actor 的引入，并不需要 double-DQN 技巧.
```python
replay_buffer.add(s, a, r, s2, done)
states, actions, rewards, next_states, dones = replay.sample() # 数据可能来自旧 policy
with no_grad():
    next_actions = pi_target(next_states) # 这一步也是为了评分. 在 285 HW3 中，这里使用了 online_actor，因此一共是3个网络

# 更新 critic

critic_optim.zero_grad()

# 软更新 target networks (e.g. tau = 0.999)
```

7. 为什么 deep Q-learning 难收敛的数学理解: 表格型 q-learning 可以直接执行 `Q <- BQ`，此处 B 为贝尔曼最优算子，其为 $L_oo$（最大同维差）上的压缩映射，全局最大误差一定乘 gamma 会不断缩小. 但多了神经网络以后多了投影算子 `Q <- ΠBV`，即在神经网络能够表达的函数集合 Ω 中找一个 V_ϕ 使得 V_ϕ 和 BV 的 L2 误差最小. 而 `ΠB` 不再是压缩映射，导致 DQN 天然不稳定.

## Lec9 Off-policy policy gradient

注意：AC 其实一直使用的是 wrong advantage 和 wrong state distribution，具体查看 [10-actor-critic算法](../hrl/10-actor-critic算法.md) 的 on-policy 注释.

1. IS: 考虑离散动作 on-policy actor-critic，要改为 off-policy 则从 replay buffer 中取出用于更新时额外乘 `pi_new(a|s) / pi_old(a|s)` 即可. 但这里有 BUG: 如果 `pi_old(a|s)` 接近 0，权重会爆炸.
2. PPO (Proximal 近端 Policy Optimization): 裁剪重要性权重到 $[1 - epsilon, 1 + epsilon]$ 范围内，例如 `0.8 ~ 1.2`. 此外，PPO 还采用了 GAE loss 以及熵正则 loss（鼓励策略多探索）.
   ```python
   # TD loss: 单步即可
   target = r + gamma * V(s2) * (1 - done)
   loss = mse(V(s), detach(target))

   # 蒙特卡洛 loss: 需要整条轨迹倒序递推
   G_t = r_t + gamma*r_{t+1} + gamma^2*r_{t+2} + ...
   loss = mse(V(s_t), detach(G_t))

   # GAE loss: 也需要整条轨迹倒序递推 (A_{t+1})
   # λ≈0.95，用于平衡 bias 和 variance, lambda 大则 variance 高 bias 小.
   delta_t = r_t + gamma * V(s_{t+1}) - V(s_t)
   A_t = delta_t + gamma*lambda*A_{t+1}
   loss = mse(V(s_t), detach(A_t + V(s_t)))
   ```

## Lec10 Advanced Policy Gradient
1. 关于上面提到的 PPO，我们还希望让训练更稳定。
    1. 加一个 KL loss(在输出的 action prob 上，KL(π_old ‖ π_new)），注意不是 entropy.
    2. 直接限制新旧 action prob 变化概率比值在 $[1 - epsilon, 1 + epsilon]$ 范围内. 这个更常用. 但 PPO 是在网络参数上更新的，怎么限制输出概率变化的? 其实并不能硬限制参数，而是在 importance sampling 中限制，避免奖励过大.

```python
# 采样时保存：old_log_probs = log(pi_old(states)[actions])
states, actions, rewards, next_states, dones, old_log_probs = replay.sample()
td_target = rewards + gamma * V_phi(next_states) * (1 - dones)
td_delta  = td_target - V_phi(states)
advantages = detach(td_delta)
log_probs = log(pi_theta(states)[actions])
ratio = exp(log_probs - old_log_probs) # ratio 即为 pi_theta(a|s) / pi_old(a|s)
unclipped = ratio * advantages
clipped   = clip(ratio, 1 - epsilon, 1 + epsilon) * advantages
actor_loss = -mean(min(unclipped, clipped))
critic_loss = mse(V_phi(states), detach(td_target))
```
